# Kex への貢献

Kex に興味を持っていただきありがとうございます！

## リリースフロー

このプロジェクトは **Tagpr** を使用してリリースを自動化しています。

1.  **プルリクエスト**: PR が `main` にマージされると、GitHub Actions ボットによって「リリース PR」が自動的に作成（または更新）されます。この PR にはバージョンの更新と `CHANGELOG.md` の更新が含まれます。
2.  **リリース**: 「リリース PR」をマージすると、新しい git tag が作成され、**GoReleaser** が自動的にバイナリをビルドして GitHub Release を公開します。

## テストガイドライン

`kex` CLI の信頼性は極めて重要です。

### 哲学
1.  **テストファースト**: テストは常にパスし、プロジェクトは常にビルド可能でなければなりません。
2.  **メンテナンス優先**: 既存のテストのメンテナンスは、新しいコードを書くことと同じくらい重要です。

### テストの実行

E2Eテストを実行するには `Makefile` ターゲットを使用します:

```bash
make e2e
```

### プレコミットチェック (Lefthook)

コミット前に品質を強制するために [Lefthook](https://github.com/evilmartians/lefthook) を使用しています。

```bash
make init
```
