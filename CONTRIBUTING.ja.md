## リリースフロー

本プロジェクトでは、**Tagpr** を使用してリリースフローを自動化しています。

1.  **プルリクエスト**: PR が `main` ブランチにマージされると、GitHub Actions ボットによって「リリース PR」が自動的に作成 (または更新) されます。この PR にはバージョンの更新と `CHANGELOG.md` の更新が含まれます。
2.  **リリース**: 作成された「リリース PR」をマージすると、新しい git tag が作成され、**GoReleaser** が自動的にバイナリをビルドして GitHub Release を公開します。

## テストガイドライン

`kex` CLI の信頼性は極めて重要です。

### 哲学
1.  **テストファースト**: テストは常に成功し、プロジェクトは常にビルド可能な状態を保つ必要があります。
2.  **メンテナンス優先**: 既存のテストをメンテナンスすることは、新しい機能を実装することと同じくらい重要です。

### テストの実行

E2Eテストを実行するには `Makefile` ターゲットを使用します:

```bash
make e2e
```

### プレコミットチェック (Lefthook)

コミット前に品質基準を担保するため、[Lefthook](https://github.com/evilmartians/lefthook) を使用しています。

```bash
make init
```
